    类 EditorViewer

    编辑器查看器

    EditorViewer 是专门为编辑场景设计的查看器，提供两种编辑模式：

    • 模型编辑模式 (MODEL_EDIT)：编辑单个模型的显示效果和SKU
    • 场景编辑模式 (SCENE_EDIT)：组合多个模型创建场景

    核心特性：

    • 状态管理：维护当前编辑模式（idle/model/scene）
    • 统一API：相同的API在不同模式下自动路由到正确的处理逻辑
    • 模式切换：流畅的模式切换和状态清理
    • 事件通知：模式变化时派发事件

    使用场景：

    // 创建编辑器
    const editor = new EditorViewer(container);

    // 模型编辑模式
    const model = await editor.loadModel({ urlOrId: 'M3EF684585' });
    editor.enterModelEditMode(model);
    editor.modelEditor.createSKU('黑色款');

    // 场景编辑模式
    editor.enterSceneEditMode();
    const modelA = await editor.loadModel({ urlOrId: 'M3EF684585', position: [0, 0, 0] });
    const modelB = await editor.loadModel({ urlOrId: 'M9876543', position: [2, 0, 0] });

    • dispose(): void

      • 事件监听器

        urlOrId: 'M3EF684585',
        position: [0, 0, 0],
        skuId: 'sku_black',
        autoSelect: true
      });
      // 自动添加到场景，应用变换，选中对象

    switchSku

    • switchSku(skuId: string): Promise<ModelObject>

      SKU切换（编辑器统一入口，封装所有同步逻辑）

      此方法封装了SKU切换的完整流程：

      1. 切换前写回当前SKU材质（避免丢失编辑）
      2. 调用底层switchSKU（换glb或应用材质）
      3. 统一后处理：更新target、状态、快照
      4. 发送事件通知UI刷新

      参数

      • skuId: string

        目标SKU ID

      返回 Promise<ModelObject>

      Promise 切换后的模型对象

      示例

      // 前端只需一行
      await editor.switchSku('SKU456');
      // SDK自动处理所有同步，UI监听事件刷新
